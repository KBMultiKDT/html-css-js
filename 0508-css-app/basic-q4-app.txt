정적 데이터:

특징:
정적 데이터는 서버에 미리 저장되어 있고, 요청이 있을 때마다 동일한 내용을 반환합니다.
주로 HTML, CSS, JavaScript, 이미지, 비디오 파일 등이 포함됩니다.
데이터는 변경되지 않고 고정되어 있습니다.
클라이언트의 요청부터 서버의 응답까지 과정:

클라이언트가 웹 페이지를 요청합니다.
서버는 요청된 페이지의 URL을 확인하고 해당하는 정적 파일을 찾습니다.
서버는 찾은 파일을 클라이언트에게 반환합니다.
클라이언트는 받은 파일을 브라우저에 렌더링하여 사용자에게 보여줍니다.

-----------------------------------------------------------
동적 데이터:
특징:

동적 데이터는 요청 시에 서버에서 생성되거나 처리되어 클라이언트에게 반환됩니다.
주로 데이터베이스 조회 결과, 사용자가 입력한 정보, 현재 시간 등이 포함됩니다.
데이터는 요청마다 다를 수 있습니다.
클라이언트의 요청부터 서버의 응답까지 과정:

클라이언트가 웹 페이지를 요청합니다.
서버는 요청을 받고, 필요한 동적 데이터를 생성하거나 처리합니다.
동적 데이터가 생성되거나 처리된 후, 서버는 완성된 페이지를 클라이언트에게 반환합니다.
클라이언트는 받은 페이지를 브라우저에 렌더링하여 사용자에게 보여줍니다.

-----------------------------------------------------------
비교:
정적 데이터는 서버에 미리 저장되어 있고, 요청마다 같은 내용을 반환합니다. 반면에 동적 데이터는 요청에 따라 서버에서 생성되거나 처리되어 다른 내용을 반환할 수 있습니다.
정적 데이터는 일반적으로 서버의 부하가 적고 응답 시간이 빠릅니다. 반면에 동적 데이터는 데이터 생성이나 처리에 시간이 걸릴 수 있어 응답 시간이 상대적으로 느릴 수 있습니다.
정적 데이터는 주로 내용이 변경되지 않는 웹 페이지의 요소에 사용되며, 동적 데이터는 사용자의 입력이나 시간에 따라 변하는 데이터에 사용됩니다.
